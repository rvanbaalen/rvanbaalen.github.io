## When to use

Use when you need a thinking partner who pushes back. The Cofounder agent enforces prioritization, challenges assumptions, and holds you accountable. It grounds itself in your actual business context on first run, then gets sharper the more you use it. Useful for check-ins, sparring on ideas, reviewing progress, or doing focused business work.

## How it works

1. On launch, the orchestrator generates a project-scoped ID from your working directory so multiple cofounder instances can coexist across projects.
2. If this is your first run, it triggers setup and onboarding to understand your business, goals, and primary metric.
3. If a legacy (unscoped) config exists, it automatically migrates your data to the new project-scoped location.
4. Once configured, it validates your workspace structure and spawns the cofounder agent with full business context.
5. From there you can spar on ideas, run cadence-based check-ins, create action briefs, or review progress and priorities.

## Getting started

```
/plugin install cofounder@rvanbaalen
/cofounder
```

First run triggers onboarding automatically.

## What's new in v1.1

- **Per-project scoping.** Config and workspace data are now scoped by project directory, so multiple cofounder instances can coexist on the same machine without conflicts.
- **Automatic legacy migration.** Existing unscoped configs are detected and moved to the new project-scoped location on first run.
- **Improved config validation.** The orchestrator now verifies the workspace structure before spawning the agent and offers clear recovery options if anything is missing.
