## Problem Statement

Robin's personal website is currently a single-page static HTML site listing open source projects in card grids with npm badges. It does not communicate who Robin is as a technology leader, has no articles/blog, no contact form for consulting inquiries, and uses a light theme with generic sans-serif typography. The site needs a ground-up redesign to serve as a personal brand site for a CEO/CTO who is available for consulting, with a modern stack, dark editorial aesthetic, and content-driven architecture.

## What Makes This Cool

The site should be screenshot-worthy through restraint, not excess. Every detail is intentional: OKLCH color system, serif typography, a subtle pulsating dot nav indicator, floating sidebar, SVG background texture. The kind of site where another developer inspects the source and finds it just as clean as the surface. "Done well IS the 10x."

## Constraints

- Must keep release-please for versioning
- Must keep Google Analytics (ID: G-DY4RKECQCF) alongside new PostHog integration (tokens TBD)
- Hosting migrates from GitHub Pages to Cloudflare Pages (enables backend functions)
- Contact form spam protection via Cloudflare Turnstile
- No framework lock-in for content — Markdown/MDX articles

- Respect `prefers-reduced-motion` for all animations

## Premises

1. **Astro is the right framework.** Content collections for articles + projects, islands architecture for interactive elements (contact form, theme toggle), static-first performance, MDX support. Vite stays as the underlying build tool.
2. **Three content types:** Articles (MDX/Markdown), projects (structured data), and page content (home, about, contact).
3. **Contact form is the primary conversion point.** Backed by Cloudflare Functions + Turnstile anti-spam. Not a footer afterthought.
4. **This is NOT a developer portfolio.** It's a CEO/CTO's personal brand site. Open source projects demonstrate technical depth and execution capability, not download counts. No npm badge walls.
5. **Dark theme with serif typography.** OKLCH color system. Understated, not braggy. The quality of what's there is the statement.
6. **Both Google Analytics AND PostHog.** GA stays, PostHog added alongside (tokens to be provided).
7. **DESIGN.md as the design system source of truth.** Colors, typography, spacing, components, animation specs.
8. **Stylish CSS transitions and animations** where applicable, always behind `prefers-reduced-motion` media queries.

## Approaches Considered

### Approach A: Ship Fast
Astro with Content Collections for articles, JSON data file for projects, single serif font, minimal components. Contact form via Cloudflare Function + Turnstile. No MDX, no tag system, no OG image generation. Clean dark theme but fewer design tokens.

Effort: S | Risk: Low
Pros: Fast to ship, easy to maintain
Cons: May feel generic, limited article features, design system implicit not codified
Completeness: 5/10

### Approach B: Elegant Craftsman
Full Astro site with typed Content Collections for articles AND projects. MDX support. DESIGN.md as codified design system. Serif + monospace font pairing. Article tags, reading time, RSS feed, sitemap, OG meta. Cloudflare Functions for contact + Turnstile. Proper 404. Dark theme with OKLCH color system. CSS transitions and animations. Every spacing value intentional.

Effort: M | Risk: Low
Pros: Matches the "elegant craftsman" vision exactly, extensible, the site itself demonstrates craft
Cons: More upfront work
Completeness: 9/10

### Approach C: Editorial Scroll
Same backend as B, but the home page is a single long-form editorial scroll — a magazine feature that IS the site. More cinematic, more memorable, but harder to maintain.

Effort: L | Risk: Medium
Pros: Genuinely unique, screenshot-worthy by design
Cons: Harder to add projects, content updates require design thinking
Completeness: 8/10

### Animations & Transitions

All animations respect `prefers-reduced-motion: reduce`.

- **Nav dot indicator:** Pulsating animation on active dot (subtle scale + opacity cycle, ~2s infinite)
- **Article cards:** Border-color and box-shadow transition on hover (0.2s ease)
- **Project rows:** Background transition on hover (0.15s ease), name color shift to accent
- **Page transitions:** Staggered fade-in on scroll for article cards and project rows (intersection observer, 0.3s delay between items)
- **Availability badge:** Pulsating green dot glow (2s infinite)
- **Send button:** Background brightens on hover (accent -> accent-bright, 0.15s)
- **Sidebar links:** Color transition on hover (0.15s)
- **Dot pattern:** Subtle parallax on scroll (CSS transform, very restrained)

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

### Background Pattern

SVG dot grid, 32px spacing, 0.6px radius circles at 3% white opacity. Masked with radial gradient (ellipse at 20% 15%, fading to transparent at 60%). Fixed position, behind all content.

### Responsive / Mobile

On viewports below 768px, the sidebar collapses into a bottom navigation bar (fixed to bottom of screen) with icon-only nav items. The photo, name, bio, and social links move into a slide-out drawer triggered by a hamburger icon. The main content area becomes full-width with reduced horizontal padding (24px).

### Navigation Style

**Dot indicator pattern (selected):**
- No background, no borders
- Text only with Phosphor icon + label
- Active state: accent-colored text + small pulsating dot (4px) positioned left of the item
- Hover: text brightens to primary color, dot appears dimmed
- Padding-left: 20px to accommodate dot space

## Deployment & CI/CD

### Migration: GitHub Pages -> Cloudflare Pages

**Staged cutover plan** (to avoid downtime):

1. Connect the repo to Cloudflare Pages, deploy to a staging URL (e.g., `rvanbaalen-github-io.pages.dev`)
2. Build command: `npm run build`, output directory: `dist/`
3. Verify the staging deployment is fully functional
4. Add custom domain `robinvanbaalen.nl` in Cloudflare Pages dashboard
5. Remove the custom domain from GitHub Pages settings (GitHub repo Settings > Pages)
6. Update DNS CNAME record to point to Cloudflare Pages (`<project>.pages.dev`)
7. Wait for Cloudflare SSL certificate provisioning (usually < 15 minutes, can take up to 24h)
8. Verify live site on robinvanbaalen.nl with HTTPS
9. Keep GitHub Actions release-please workflow (creates releases, triggers Cloudflare Pages auto-deploy on push to main)

**Note:** Steps 5-7 involve a brief DNS propagation window where the site may be unreachable. Schedule this during low-traffic hours. The staging URL in step 1 serves as a fallback to verify the build works before cutting over.
